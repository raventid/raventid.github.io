In this article I want to give you some answers to the WHY? questions. Why we need TLA+, what problems are extremely suitable for TLA+ computational model. TLA+ is a tool to build and verify models for your software systems. It is suitable not only for software systems, but basically for any type of systems. And in the following articles I will talk about modeling network level algorithms for network packet transfer, modeling analogues clock, and modeling life and death. Why is it possible with TLA+? Because it is not just a programming language, TLA+ is a set of tools that includes a notation (aka. express specification in a text form). Checker and IDE. My suggestion for learning TLA+ is the same as for learning a new foreign language (or a new paradigm in programming languages) very different from yours. Try to be open-minded, because writing specification is closer to writing mathematical formulas, and not writing the computer programs, so you will have to work through a somewhat steep learning curve, but I will take your hand and guide you through the whole process. To be precisie TLA+ still has some resemblense to functional programming language, but don't let this resemblense lead you in the wrong way and make you think that you are writing a program. You don't. You describe the behaviour of the system in time. Now, we will test your TLA+ setup and write some simple specification just to see if we are ready to move forward. Let's create a new specification and write this

module Clock vars hour Next == hour' = 3 Spec == [][Next]_hour
